Two layers, one binary

abyss is a code intelligence engine AND a token compressor. They reinforce each other.

🔍

Code Intelligence

Call graph, blast-radius analysis, hotspot scoring, change coupling — all from tree-sitter + git history. No language server needed.

⚡

Proxy Compression

Intercepts agent commands (git, cargo, grep, cat…), structurally compresses output, delivers 70-99% fewer tokens. Never-worse guard ensures zero information loss.

📈

Semantic Enrichment

Compressed output is annotated with blast-radius warnings, hotspot scores, and change coupling — signals no pure compressor can provide.

🌳

Tree-sitter Smart Read

AST-level body stripping for 14 languages. cat on a 600-line file returns 98 lines of signatures. Brace-counting fallback for the rest.

🔒

Pre-edit Safety Cards

Before every edit, agents see who calls the function, blast radius, test coverage gaps, and change-coupled files. 2ms hook latency.

⚙

One-command Setup

abyss setup — indexes, installs hooks into Claude/Codex/Gemini, enables proxy. Zero config files, zero dependencies.

vs. pure compressors

abyss is not "another RTK" — it's a code-aware compression layer.

Pure compressor (e.g. RTK)

  • Pattern-matching output filters
  • 100+ command handlers
  • 14 AI tool integrations
  • No code understanding
  • No blast-radius annotations
  • Brace-counting smart read
  • Requires separate install

abyss proxy

  • Structural + semantic filters
  • 22 handlers + TOML engine
  • Claude / Codex / Gemini hooks
  • Call graph + impact analysis
  • Blast-radius & hotspot annotations
  • Tree-sitter AST body strip (14 langs)
  • Built into existing binary

Full CLI surface

Everything is a subcommand. No plugins, no runtime dependencies.

# Code intelligence
abyss index Build the structural index
abyss callers Symbol Who calls this?
abyss impact Symbol What breaks if I change it?
abyss context file Full pre-edit context
abyss map Hotspots, coupling, risk areas
abyss where file Architectural coordinates

# Proxy compression
abyss proxy <cmd> Run + compress + track
abyss proxy --explain Show which handler fired and why
abyss gain Token savings dashboard

# Integration
abyss setup One-command onboarding
abyss attach claude Install hooks (proxy on by default)
abyss mcp MCP server (stdio)
abyss watch Live reindex on file save

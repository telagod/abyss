# Dogfood: FastAPI v0.115.4 ‚Äî MRO L0e validation

**Date**: 2026-06-17
**Target**: `fastapi/fastapi @ tag 0.115.4` (~50K Python LOC, class-based DI)
**Binary**: `abyss 0.4.0` (slim, release build, main HEAD `116bc52`)
**Hypothesis under test**: E3's Python MRO walker (L0e) ‚Äî predicted ‚â•50 hits on FastAPI vs 2 on click.

## TL;DR

> L0e fired **0 times** on FastAPI v0.115.4. Hypothesis falsified.
>
> Root cause is structural, not a bug: FastAPI's class hierarchy roots all
> sit in **external libraries** (Starlette, pydantic, stdlib). The L0e walker
> can only resolve methods when the inheritance chain stays in-repo. Out of 345
> unresolved typed-receiver calls (the L0e candidate pool), 100% dead-end at
> external bases on the first hop. click's 2 hits were comparatively rich.

**click was the wrong baseline.** FastAPI doesn't define a deep in-repo class
hierarchy ‚Äî it bridges into Starlette/pydantic. The MRO walker as designed
helps real-world repos that re-implement inheritance internally; pure web
frameworks that subclass third-party libs benefit from a **different**
mechanism (stub-class registry or vendored type metadata).

## Bugs surfaced

### B1 ‚Äî F4's test-path filter misses top-level `tests/`  (HIGH)

**Pattern**: F4 (commit `54ef34a`) added `f.path NOT LIKE '%/tests/%'` to L4
and L4b. On FastAPI, all 537 test files live at `tests/test_*.py` (top-level)
and DON'T match `%/tests/%` because the leading `%` requires a slash before
`tests/`. Verified directly: `SELECT COUNT(*) FROM files WHERE path LIKE
'%/tests/%'` ‚Üí 0, while `LIKE 'tests/%'` ‚Üí 537.

**Evidence**: `fastapi/applications.py:913` references `State` ‚Üí resolved to
`tests/test_router_events.py` at confidence 0.8 (L4 global-unique). F4 was
supposed to prevent this exact shape.

**Suggested fix**: add `f.path NOT LIKE 'tests/%' AND f.path NOT LIKE 'test/%'`
(start-anchored, no leading slash) to `NOT_TEST_PATH` in `src/indexer/pipeline.rs:508`.
Add a regression test using `fastapi/applications.py:913 ‚Üí State` as a fixture.

### B2 ‚Äî `docs_src/` (tutorial samples) treated as production candidates  (HIGH)

**Symptom**: `fastapi/dependencies/utils.py:459` resolves `Query` to
`docs_src/graphql/tutorial001.py` at confidence 0.5 (L5 ambiguous) instead of
the real `fastapi/param_functions.py`. Three `Query` definitions exist; the
ambiguity picker has no preference for `fastapi/` over `docs_src/`.

**Suggested fix**: add `f.path NOT LIKE 'docs_src/%'` and `f.path NOT LIKE
'docs/%'`, or generalize: introduce a `NOT_NON_PROD_PATH` filter family applied
in L4/L4b/L5 covering sample/tutorial/example directories. Even better:
a per-language convention table (`examples/`, `samples/`, `docs_src/`, `book/`
for Rust mdbook) that filters candidate symbols when the source is in
production-shaped paths.

### B3 ‚Äî Arch labeller pulls hub files into the docs_src cluster  (MEDIUM)

**Symptom**: `fastapi/applications.py` labeled `module=docs_src` because 684
docs_src files reference it. Connectivity clustering favours the dominant
referring side rather than the file's own path. `fastapi/routing.py` similarly
clustered with tests as `cluster-2 (test)`.

**Suggested fix**: when a file's own path matches a prod prefix (`fastapi/`,
`src/`, root pkg name from pyproject.toml/Cargo.toml) AND its module cluster's
centroid_path doesn't, override the module assignment to a path-derived module.
Or: weight module clustering by edge direction ‚Äî referring-in edges from
docs_src/tests/ should not dominate.

### B4 ‚Äî L0e cannot cross into external libraries  (DESIGN GAP)

**Symptom**: L0e=0 on FastAPI despite 345 candidate refs. All inheritance
chains exit the repo on the first hop into Starlette/pydantic/stdlib.

This isn't strictly a bug ‚Äî it's the documented design boundary ("V1
approximation; qualified bases like `click.Command` are unresolvable to a repo
file and skipped"). But on web-framework-style repos the design boundary makes
L0e a no-op. Two paths forward:

- **(cheap)** A stub registry ‚Äî hand-curated `class Starlette { add_route,
  add_middleware, add_exception_handler, ... }` virtual symbols indexed at
  startup for the top 10 Python framework deps. Confidence 0.7 (not 0.95)
  because no source verification.
- **(expensive)** When `pyproject.toml` declares deps, scrape methods from the
  installed venv site-packages and index them as `external=1` symbols. Lets
  L0e walk past the boundary.

Track as a separate epic; **don't** rush a partial fix.

### B5 ‚Äî `--depth 1` clone hides hotspots  (KNOWN, low priority)

Shallow clone has no commit history ‚Üí `coupling.rs` correctly suppresses,
`hotspot.rs` falls back to complexity-only. Surfaced clearly in the `map`
hint added in commit `116bc52`. No action.

### Comparison: click vs FastAPI L0e

| Repo | LOC | L0e hits | Candidate pool | Saturation |
|---|---|---|---|---|
| click 8.x | ~10K | **2** | small (mostly in-repo `Command`/`Group`) | 2/2 hit |
| FastAPI 0.115.4 | ~50K | **0** | 345 | 0/345 hit |

**The 100√ó prediction was the wrong direction.** Click has fewer candidates
but ALL stay in-repo; FastAPI has 100√ó more candidates but they ALL exit the
repo. L0e measures in-repo inheritance complexity, not codebase size or DI
density. Need a different probe (e.g., scikit-learn, Django ORM internals,
SQLAlchemy core) where deep inheritance stays inside the package.

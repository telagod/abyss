## 5. Bugs / UX debts discovered

1. **`callers <Type>` is blind to TypeScript interfaces and type aliases.** 75 resolved refs to `ViteDevServer` exist at confidence 0.95, all with `kind='type_ref'`. The `callers` command filters to `kind='call'` only, so it returns "no callers found" for a heavily-used exported interface. This is the single biggest UX gap for TS/JS codebases. Fix: when the target symbol's kind is `interface`/`type`/`class`, include `type_ref` rows; or ship a sibling `references` command.

2. **arch_map labeller still degraded on monorepos.** vite's modules come back as `p-2`, `p-3`, … `p-18` (15 of 55 modules) and three `mixed:create-vite+-0` style labels. The labeller is picking up `packages/` and truncating to `p`, then disambiguating by id. Same root cause as helix's `helix--N` issue but worse here (vite's deeper directory tree exposes the prefix-cut path more often). Fix: when the module's centroid path is `packages/{X}/...`, drop the `packages/` segment and use `{X}/...`.

3. **`mixed:` prefix labels surface raw cluster-merge state.** Three modules are named `mixed:create-vite+-0`, `mixed:create-vite+`, `mixed:hmr-ssr+`. The `mixed:` prefix and trailing `+`/`-0` are internal cluster-merge bookkeeping leaking into a user-visible field. Either resolve to a single dominant label or relabel as `cluster-N` to signal "we couldn't name this".

4. **Contracts section duplicates symbols with identical caller counts.** `waitForRequestsIdle` appears twice in the `server/index.ts` card — same caller count (1 + 0 tests) both times. Same root cause as helix's `default (method)` 18× repetition: bare symbol name collisions across multiple definitions collapse to identical rows. Render as `Foo.method` or `module::name` to disambiguate.

5. **L4 global-unique mis-resolves common library names to test fixtures.** `context resolve.ts` shows `debug → playground/hmr-ssr/__tests__/hmr-ssr.spec.ts`. The `debug` import in `resolve.ts` is the npm package; abyss resolved it via L4 global-unique to a same-named local function in a test file. Likely fix: when L4 finds a candidate inside `__tests__/`, `playground/`, or `*.spec.ts`, prefer leaving unresolved over resolving to it.

6. **`map` command output dominated by hotspots heading even when empty.** On shallow clones `map` prints `═══ Hotspots ═══` and stops — the user sees no message explaining why. The `--json` output (`{"coupling":[],"hotspots":[]}`) is more honest. Suggest printing `(no hotspot data: insufficient commit history)` when the table is empty.

7. **Module label `p-N` provides zero signal in user-facing output.** Same finding as helix `helix--N`. When the cluster has no good label, the output should say so (`unlabelled-18` or `cluster-18`) rather than appending a digit to a one-letter prefix-cut. An agent reading `module=p-18` cannot do anything with it.

8. **High unresolved fraction (68%) on TypeScript.** Likely a mix of (a) bare imports from external npm packages (legitimately unresolvable — abyss has no node_modules), (b) type-position references where the resolver doesn't bind type imports to type definitions, (c) the same `type_ref` blindness as bug 1. Some of (b) and (c) are recoverable.

None of these are correctness regressions on the parts that do resolve — resolved refs are high-confidence and accurate. The bugs are all in coverage (interfaces invisible to `callers`), labelling (`p-N`, `mixed:`), and one resolver false-positive (`debug → test fixture`).

## 6. Final verdict: **7 / 10**

abyss handled a 1,793-file polyglot TS/JS monorepo in **863 ms cold, 172 ms warm**, zero crashes, zero degraded-mode warnings, p95 hook latency under 20 ms — the structural pipeline is solid and the cross-language family filter holds (0 truly cross-family edges in 28K refs). The two probes that matter most to a vibecode agent — `context` (4 external entry points with call sites named) and `impact` (4 prod entry pathways + 14 uncovered paths + 7.4/10 risk score) — deliver actionable signal a grep cannot. But the headline gap is **`callers <Interface>` returns nothing on TypeScript types** because the `kind='call'` filter excludes the 75 high-confidence `type_ref` rows that exist for `ViteDevServer`; this makes the single most common agent question on TS codebases ("who uses this exported type?") unanswerable, and is the highest-ROI fix surfaced by this dogfood. Secondary gaps — monorepo labeller still emits `p-N` / `mixed:` placeholder modules, L4 occasionally mis-resolves `debug` to a test fixture, contracts list duplicates bare-name symbols — are all in presentation/coverage, not correctness. Net: an agent editing vite would benefit from abyss right now (the cards, blast radius, and impact analysis are all real signal), but a TS-aware `references` command and a monorepo-aware labeller would move this from 7 to 9.
